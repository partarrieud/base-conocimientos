# Study Order

Roadmap de 12 meses orientado a consolidar una base fuerte en backend .NET, datos, arquitectura, cloud y preparacion de entrevistas tecnicas para LATAM.

## Enfoque general

- Dedicar entre 5 y 7 horas semanales.
- Combinar teoria, ejemplos practicos y narrativa de entrevista.
- Cerrar cada mes con un repaso y una simulacion de respuestas habladas.
- Documentar decisiones, incidentes y aprendizajes usando las plantillas del repositorio.

## Mes 1 - Fundamentos

- Dependency Injection, Middleware, manejo de errores y configuracion en ASP.NET Core.
- Async/Await, CancellationToken y principios de concurrencia.
- SOLID, separacion de responsabilidades y diseño orientado a mantenibilidad.
- Practica de respuestas base de Senior .NET.

## Mes 2 - Backend

- Entity Framework Core, Dapper y criterios para elegir entre ambos.
- CQRS, MediatR, validaciones y pipeline behaviors.
- Caching, logging estructurado y health checks.
- Disenar una API REST empresarial simple y documentar trade-offs.

## Mes 3 - Bases de datos

- Indexes, Execution Plans, Statistics y Query Tuning.
- Transactions, Isolation Levels, locking, blocking y deadlocks.
- TempDB, parameter sniffing y estrategias de troubleshooting.
- Practica de preguntas de Senior SQL.

## Mes 4 - Frontend

- Hooks, lifting state, renderizado y diseño de componentes.
- State Management y performance optimization.
- SPA vs SSR y criterios para decisiones de rendering.
- Preparar respuestas de React con foco en integracion con APIs REST.

## Mes 5 - Arquitectura

- DDD, Clean Architecture y Hexagonal Architecture.
- Monolith vs Microservices con ejemplos reales.
- Resilience Patterns, Circuit Breaker e idempotencia.
- Documentar un caso de arquitectura con trade-offs explicitos.

## Mes 6 - Cloud

- Azure App Service, Azure Functions y Azure Storage.
- Azure Service Bus vs Kafka segun contexto.
- Managed Identity, Key Vault y secretos.
- Practicar narrativa de despliegue seguro y operacion en Azure.

## Mes 7 - Kubernetes

- Docker, multi-stage builds y networking.
- Kubernetes basico: Pod, Deployment, Service, Ingress.
- ConfigMaps, Secrets, probes y rollout strategies.
- Armar una historia de despliegue realista end-to-end.

## Mes 8 - Kafka

- Producers, Consumers, Partitions y Consumer Groups.
- Ordering, retries, DLQ, idempotencia y exactly once.
- Schema Registry, Avro y evolucion de contratos.
- Disenar un flujo distribuido con integridad razonable.

## Mes 9 - System Design

- Escalabilidad horizontal, caching y consistencia eventual.
- Outbox Pattern, Sagas y observabilidad distribuida.
- Diseñar sistemas con requerimientos no funcionales claros.
- Practicar whiteboard verbal: APIs, datos, colas, despliegue y monitoreo.

## Mes 10 - Leadership

- Priorizacion tecnica, deuda tecnica y manejo de riesgos.
- Comunicacion con negocio, producto y otros equipos.
- Mentoria, definicion de estandares y revision de diseño.
- Preparar historias STAR con foco en impacto concreto.

## Mes 11 - Entrevistas

- Simulacros por stack: .NET, SQL, React, Arquitectura.
- Simulacros de System Design y liderazgo tecnico.
- Refinar respuestas cortas, medianas y profundas para cada tema.
- Ajustar ejemplos segun tipo de empresa: producto, consultora o fintech.

## Mes 12 - Consolidacion

- Repaso global de `P1` y `P2` de la lista priorizada.
- Completar vacios tecnicos detectados en simulacros.
- Actualizar proyectos reales, postmortems y aprendizajes.
- Dejar armado un resumen corto de repaso pre-entrevista.

## Fases transversales

### Backend

- Mantener un hilo conductor centrado en APIs REST, integracion, observabilidad y performance.

### Bases de datos

- Relacionar cada concepto con latencia, concurrencia, throughput y consistencia.

### Frontend

- Enfocar React desde el impacto en experiencia, mantenibilidad y performance.

### Arquitectura

- Justificar decisiones desde trade-offs, no desde modas.

### Cloud

- Relacionar costo, seguridad, operacion y velocidad de entrega.

### Kubernetes

- Entender despliegue, configuracion, disponibilidad y troubleshooting.

### Kafka

- Entender semanticas de entrega, orden, escalado y contratos.

### System Design

- Poder explicar requerimientos, limites, cuellos de botella y mitigaciones.

### Leadership

- Mostrar criterio, influencia, ownership y claridad para priorizar.

### Entrevistas

- Convertir conocimiento tecnico en respuestas concretas, ordenadas y convincentes.
