# .NET

Area central del repositorio para backend, APIs REST y decisiones de implementacion sobre la plataforma .NET.

## Subareas

- `Basics`: conceptos base que conviene dominar para explicar el stack con claridad.
- `Core`: runtime, concurrencia, pipeline HTTP y componentes transversales.
- `Data`: acceso a datos y criterios de persistencia.
- `Architecture`: patrones de aplicacion y organizacion interna.
- `Advanced`: rendimiento, memoria y optimizacion.
- `Testing`: notas especificas sobre testing en ecosistema .NET.

## Prioridades recomendadas

1. Dependency Injection, Async/Await y Middleware.
2. EF Core, Dapper y criterios de acceso a datos.
3. CQRS, caching, Background Services y performance.
4. Garbage Collector, manejo de memoria y optimizacion fina.

## Como estudiar esta carpeta

- Arrancar por los temas `P1` si el foco es entrevista inmediata.
- Relacionar cada concepto con APIs REST y con problemas de produccion.
- Practicar respuestas cortas y luego profundizar con ejemplos reales.
