# SQL Server

Area orientada a modelado, tuning y troubleshooting sobre SQL Server con foco en entrevistas y problemas reales de produccion.

## Subareas

- `Basics`: fundamentos que aparecen siempre en entrevistas.
- `Tuning`: diagnostico y optimizacion de consultas.
- `HighAvailability`: continuidad operativa y recuperacion.

## Prioridades recomendadas

1. Indexes, execution plans, statistics y query tuning.
2. Transactions, isolation levels, locking, blocking y deadlocks.
3. TempDB, parameter sniffing, partitioning y alta disponibilidad.
